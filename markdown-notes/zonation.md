# 保护区规划
空间保护的优先次序关系到保护行动的有效分配。保护区规划包括开发基于生态价值的保护价值模型，数据预处理，空间优先级分析以及对保护行动的结果进行解释。Zonation可以根据模型逐步移除不那么重要的单元格，最终剩下那些需要优先保护的单元格。
## 保护区区划的理论依据
### 附加效益函数ABF
附加效益函数通过特定特征的物种的面积变化曲线对保护区进行规划，可以简单理解为优先保护搞物种丰富度的区域。
### 核心区区划CAZ
CAZ根据单元中某个最重要特征出现的次数对单元进行排名，可以简单理解为优先保护有着高价值物种（濒危物种）的区域。
ABF和CAZ并不是二元对立的，ABF强调内部区域的丰富性，而CAZ则是对某类特殊功能进行重点保护。
## 关于区划的物种分布数据
1. 连续数据，可以是存在/不存在这样的，也可以以分布概率为结果进行输出的数据（如MaxEnt的输出）
2. 分类数据，某些土地利用或者生境类型是这种样式的数据，但是也可以把其中每一种类型单独提取出来作为一个个的0/1数据以方便修改参数。
3. 点分布数据，不建议，因为模型会将优先级集中在点观测数据上，而且连通性要素也往往不能应用在点观测数据上。
Tips：可以使用ArcGIS的模型构建器等工具对数据进行预批处理。
## 运行模型
运行模型需要三种文件：
1. 项目文件（输入与输出）
2. 生物多样性特征清单（每个特征有参数、权重、连通性等指标）
3. 设置文件（如何进行分区优先级排序）
### 项目文件
在txt进行编辑并将后缀改为.bat的脚本即可
```
call zig4.exe -r input/settings.dat input/biodiversityfeatures.spp output/output.txt 0.0 0 1.0 0
```
### 特征清单
就是上文中出现的`biodiversityfeatures.spp`文件，.spp后缀，每行对应一个特征，每个特征有六列。
```
4.0 0.002 -1 -1 0.25 tiger.img
1.0 0.002 -1 -1 0.25 glassland.img
```
其中第1列是特征权重，比如物种的濒危等级，该指标主要用于基于CAZ的保护区规划
第2至4列为连接设置，其中第3，4列为BQP连通性响应的整数参数，通常不使用，可以给一个伪值-1。第二列用的比较多，是距离扩散递减核的十进制参数，如果分布层文件中的距离单位是**米**，那么上面的值表示每**公里**的平均扩散距离为0.002。
根据所使用的单元格移除规则，第五列也比较有用，如果是基于ABF进行保护区规划，那么该参数控制生物多样性特征对栖息地丧失的敏感度。该参数与经验观察到的**物种面积**曲线密切相关，也可以使用0.25的一般值进行替代。
如果是基于目标的规划（即为希望未来达到什么效果），那么第五列中的参数用于设置每个生物多样性特征达到的覆盖率目标（比如希望tiger可以覆盖50%，就填0.5）。
### 设置文件
也是个.dat的脚本：
```
[settings]
removal rule = 2
warp factor = 2000
edge removal = 1

BLP = 0.1

mask missing areas = 1
area mask file = ../../Data/Masks/landmask_100m.tif
```
其中， `removal rule`为保护区区划依据，1为CAZ，2为ABF，3为TBF；
`edge removal`为是否允许从边缘移除单元；
`warp factor`为移除单元的速度（每次移除多少个）
